# Meta Skill Creator

Claude Codeでスキルを作成・更新・プロンプト改善するためのメタスキル。
**対話形式のインタビュー**でユーザーの要件を引き出し、最適なスキルを自動生成します。

[![Claude Code](https://img.shields.io/badge/Claude%20Code-Plugin-purple.svg)]()

---

## 特徴

| 特徴 | 説明 |
|------|------|
| **対話型インタビュー** | 8段階の質問で要件を明確化。曖昧なアイデアから具体的な仕様へ |
| **Collaborativeモード** | ユーザーと対話しながらスキルを共創（推奨） |
| **Orchestrateモード** | Claude Code / Codex を使い分けて最適な実行エンジンを選択 |
| **24種類のスクリプトタイプ** | API連携、データ処理、開発ツールなど幅広く対応 |
| **Progressive Disclosure** | 必要な時に必要なリソースのみ読み込み |

---

## インストール

```bash
# マーケットプレイスを追加
/plugin marketplace add daishiman/meta-skill-creator

# プラグインをインストール
/plugin install daishiman-skill-creator
```

Claude Codeを再起動して完了。

## 使い方

### 基本的な流れ

1. **スキル作成を依頼** - Claude Codeに話しかけるだけ
2. **インタビューに回答** - 8段階の質問に答える（選択肢から選ぶだけ）
3. **スキル生成** - 回答に基づいて最適なスキルが自動生成される

### スキル作成の開始

Claude Codeに話しかけるだけでインタビューが始まります：

```
「新しいスキルを作成したい」
「〇〇を自動化するスキルを作って」
「毎日のレポート作成を自動化したい」
```

または `/skill-creator` コマンドで直接呼び出し：

```
/skill-creator
```

### インタビューの流れ（8段階）

スキル作成を依頼すると、以下の順序で質問されます：

| Phase | 質問内容 | 例 |
|-------|----------|-----|
| 0-1 | **ゴール特定** | 「何を達成したいですか？」 |
| 0-2 | **機能ヒアリング** | 「必要な機能は？」 |
| 0-3 | **外部連携** | 「連携するサービスは？」（API自動推薦あり） |
| 0-4 | **スクリプト** | 「どんな処理が必要？」 |
| 0-5 | **オーケストレーション** | 「処理の流れは？（順次/並列/条件分岐）」 |
| 0-6 | **スケジュール** | 「いつ実行？（手動/定期/イベント）」 |
| 0-7 | **ドキュメント** | 「APIガイドは必要？」 |
| 0-8 | **構成・優先事項** | 「シンプル？フル機能？」 |

各質問では**選択肢が提示**されるので、選ぶだけで回答できます。
自由記述も可能なので、想定外の要件にも対応できます。

### 入力例と回答例

```
ユーザー: 「GitHubのPRを自動でSlackに通知するスキルを作って」

Claude: ゴールを確認させてください。
        1. PR作成時に即座に通知
        2. 毎日のPRサマリーを通知
        3. レビュー待ちのPRを通知
        4. その他（自由記述）

ユーザー: 「1」を選択

Claude: 次に、外部連携についてお聞きします。
        GitHub APIとSlack APIの連携が必要です。
        認証方式はどちらを希望しますか？
        1. 環境変数でトークン管理（推奨）
        2. OAuth認証
        3. その他

... (以降、順番に質問が続く)
```

## 高度な機能

### オーケストレーション

複数のスキルを組み合わせて複雑なワークフローを実現：

| パターン | 説明 | 使用例 |
|----------|------|--------|
| chain | 順次実行 | データ取得→変換→保存 |
| parallel | 並列実行 | 複数APIから同時取得 |
| conditional | 条件分岐 | 成功→ログ、失敗→通知 |

### イベントトリガー

| タイプ | 説明 | 例 |
|--------|------|-----|
| file | ファイル変更を監視 | `*.md`ファイルの変更 |
| cron | 時刻ベース | 毎日9:00 |
| webhook | 外部イベント | GitHub webhook |

### スケジューラ

Cron形式で定期実行を設定：

```
0 9 * * *     # 毎日9:00
0 9 * * 1     # 毎週月曜9:00
0 0 1 * *     # 毎月1日0:00
```

### ドキュメント自動生成

- **APIドキュメント**: エンドポイント、パラメータ、使用例を自動生成
- **セットアップガイド**: インストールから使用開始までの手順を自動生成

### 統合推奨

目的に応じて最適なAPI・ライブラリを自動推薦：

```
ユーザー: 「Slack通知機能が欲しい」
Claude: Slack連携には以下のAPIを推薦します：
        1. Slack Web API（推奨）- メッセージ送信に最適
        2. Slack Incoming Webhook - シンプルな通知向け
```
